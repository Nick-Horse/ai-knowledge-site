## 背景

客户已在 Shopify 上跑通全流程（选品→上架→营销→订单），业务验证通过。但因 **客户服务完全依赖人工**，处理不了持续增长的咨询量，业务无法放大。

需求：为 Shopify 独立站定制开发一套 **AI 智能客服系统**，对接店铺数据，自动处理常见问题，复杂问题转人工。

## 核心结论

**Shopify Basic/Starter 个人站完全支持接入自定义智能客服系统。** 所有计划（包括 $29/月的 Basic）都提供完整的 Admin GraphQL API 和 Webhooks 能力，不限制第三方系统的深度集成。

---

## 技术方案全景

### 方案一：Shopify App Store 即装即用（最快）

现有成熟插件一览：

| 产品 | 核心能力 | 价格 |
|------|---------|------|
| **Tidio** | AI Chatbot + Live Chat，自动回复+人工转接 | 免费版起 |
| **Chatty** | AI 客服，自动同步产品/订单数据 | $39/月起 |
| **Gorgias** | 高级 AI 客服，订单操作+多渠道统一收件箱 | $60+/月 |
| **Kommunicate** | AI Agent，可解决 80% 重复问题 | 有免费版 |
| **VanChat** | AI 导购+客服 二合一 | 按量计费 |

**优势**：开箱即用，一周内上线
**劣势**：数据在第三方平台，无法深度定制，无法差异化竞争

### 方案二：JS 代码嵌入 + 自有后端（推荐方案）

适合需要**深度定制**的团队。架构如下：

```
┌──────────────────┐     embed <script>      ┌──────────────────────┐
│  Shopify 店铺      │ ◄────────────────────── │  智能客服 Widget     │
│  (theme.liquid)    │                         │  (JS 嵌入前端)       │
└────────┬─────────┘                         └──────────┬───────────┘
         │                                               │
         │  Shopify Admin GraphQL API                    │  大模型 RAG 引擎
         │  (商品/订单/客户数据)                          │  (FAQ/产品文档/知识库)
         ▼                                               ▼
┌──────────────────┐                         ┌──────────────────────┐
│  Shopify 后端     │                         │  自有后端服务器       │
│  Webhooks 事件推送 │◄──── order/create ────│  (Node.js / Python)  │
│  (实时通知)       │    customer/create     │                      │
└──────────────────┘    fulfill/create       └──────────────────────┘
```

**实施步骤**：

1. 在 Shopify 后台创建私有 App，配置 API 权限，获取 Access Token
2. 在 `theme.liquid` 中嵌入自定义 JS 脚本标签
3. 自有后端通过 **GraphQL Admin API** 对接订单/商品/客户数据
4. 配置 **Webhooks** 接收新订单、退款、客户注册等实时事件
5. 前端 Widget 对接大模型 + RAG 知识库回答问题

## API 能力详解

### 认证方式

- Shopify Admin API：**OAuth 2.0**（创建私有 App 获得 `shpat_` 开头的 Access Token）
- **所有 Shopify 计划（包括 Basic $29/月）都支持**创建私有 App

### 可获取的核心数据

| 数据类型 | API 端点 | 用途 |
|---------|---------|------|
| **订单** | `orders(first: 50, query: "...")` | 查订单状态、物流、退款 |
| **客户** | `customers(first: 50, query: "...")` | 查客户信息、历史订单 |
| **商品** | `products(first: 50)` | 查商品详情、库存、价格 |
| **商品变体** | `variants` | 查 SKU、库存数量 |
| **收藏/分类** | `collections` | 商品归类 |
| **元字段** | `metafields` | 自定义数据扩展 |

> ⚠️ **注意**：Basic 计划的 **Analytics API** 受限（销售报表聚合不可用），但 `orders` / `customers` / `products` 等核心数据接口**全量开放**，智能客服完全够用。

### 实时事件推送（Webhooks）

支持 50+ 种事件类型，常见的有：

- `orders/create` — 新订单
- `orders/paid` — 已付款
- `orders/cancelled` — 取消
- `orders/fulfilled` — 发货
- `customers/create` — 新客户注册

### API 限流策略

- GraphQL Admin API 基于**查询成本（cost）**而非请求次数
- 每个 token 有 1000 cost 配额，每分钟重置
- Basic 计划的配额与 Advanced 计划**一致**，不限制数据访问范围

## 推荐实施路线（三期）

### 第一期：MVP — 智能问答上线

- 开发 JS Widget 嵌入 Shopify 主题
- 搭建 RAG 知识库（产品 FAQ、售后政策、物流规则）
- 对接大模型，自动回答 70% 常见重复问题
- 人工兜底：无法回答的转接人工客服

**预计开发周期：5-7 天**

### 第二期：数据深度对接

- 通过 Admin GraphQL API 连接订单/客户数据
- 实现上下文感知：AI 知道客户买了什么、订单到哪了
- 自动化售后：根据规则自动审批退款/补发

**应用场景示例**：
> 客户问："亲，我的包裹到哪了？"
> AI：自动查询订单号 → 调用物流 API → 回复："您的包裹已发出，EMS 单号 xx，预计后天送达。"

### 第三期：多渠道 + 多语言

- 接入 WhatsApp / Email / Instagram Messaging
- 多语言自动翻译客服（跨境卖家刚需）
- AI 主动营销：根据客户浏览行为推荐商品

## 为什么选择自建而非 App Store 插件

| 维度 | App Store 插件 | 自建方案 |
|------|---------------|---------|
| **数据安全** | 数据在第三方平台 | 自有服务器，完全可控 |
| **定制化** | 固定功能，无法深度修改 | 按需开发，灵活扩展 |
| **差异化** | 每家店铺都一样 | 可形成产品壁垒 |
| **成本** | 月费 $39-$750+ | 一次性开发 + 服务器成本 |
| **知识库** | 通用知识 | 可对接企业私有知识库 |

## 技术栈建议

- **前端**：React Widget（轻量，独立于 Shopify 主题）
- **后端**：Node.js (Express/NestJS) 或 Python (FastAPI)
- **大模型**：接入任意 LLM API（支持自有 Key）
- **知识库**：RAG 架构（向量数据库 + 文档切分）
- **对接**：Shopify GraphQL Admin API + Webhooks

---

**总结**：Shopify 平台对自建系统的开放度非常高，Basic 计划即可满足智能客服的全部数据需求。真正需要投入精力的不是平台对接，而是**业务知识库的构建质量**和**对话体验的打磨**。
